Headshots load live at read time straight from the WCM directory (directory.weill.cornell.edu) — never stored, never via the ETL. On-demand sources (RePORTER, NSF) aren't on a Step Functions cadence yet; they run via the daily prototype chain. Human Resources (employer/employee mentees) is a planned source, not yet built. POPS (the public weillcornell.org physician directory) enriches clinical scholars with board certifications, specialties, and expertise (etl/pops/index.ts) — it runs after Enterprise Directory (it keys off the clinical-profile flag ED sets) and feeds the people search index; it rides the daily chain, not yet a standalone Step Functions step. Clinical trials originate in OnCore (the CTMS — a manual institutional export, static until the next export lands) and are enriched against the ClinicalTrials.gov registry (API v2); both stage into reciterdb tables (clinical_trials / clinical_trials_enriched, the latter pulled upstream by ReciterAI) that the nightly ETL reads — SPS never calls ClinicalTrials.gov directly.

View 3 · for the cloud team

Network topology

The review-ready network picture: VPC, subnet tiers, the three default-deny security groups (SG-to-SG only), and VPC endpoints. The settled direction (2026-06-30 decision) is full consolidation: the two per-env Sps VPCs are replaced by one shared, TGW-attached its-reciter-vpc01 that hosts the whole estate — App + Data + ETL, both envs — with no VPC peering, env isolation by per-env security groups in one shared CIDR, and WCM / on-prem sources reached natively over the TGW (closing the #443 gap). The edge front door is NetScaler (View ④). Cutover is flag-gated and reversible — pending.

Settled direction — full consolidation (2026-06-30)

  • The two per-env Sps VPCs are replaced by one shared, TGW-attached its-reciter-vpc01; the whole estate (App + Data + ETL, both envs) moves in. No VPC peering.
  • Env isolation is by per-env security groups inside the one shared VPC — no network boundary, SG-to-SG only.
  • WCM / on-prem sources (ED-LDAP, ReciterDB, ASMS, COI, Jenzabar) are reached natively over the shared VPC's TGW attachment — this closes the #443 connectivity gap.
  • Edge front door = NetScaler (CloudFront + WAF → NetScaler → ALB → Fargate; View ④), inserted as a follow-on decoupled from this move.
  • Cutover is flag-gated and reversible (data moves by snapshot-restore into a fresh cluster alongside the live one) — pending. Plan: docs/sps-vpc-consolidation-plan.md.

View 4 · the edge decision — resolved (#502)

Edge topology — resolved

The edge front door is decided. Fabrice confirmed (2026-07-02) the target is CloudFront + WAF → NetScaler → ALB → Fargate: CloudFront and the WAF stay, and the AWS ALB is kept because the app is ECS Fargate. NetScaler is not in the path today — both distributions point straight at their ALB — so reaching the target is a NetScaler-insertion change, orthogonal to the item-3 VPC move. It lands as a decoupled follow-on: repoint the CloudFront origin to the new ALB now, then WCM inserts NetScaler in front as a separate step.

View 5 · inside the app (C4 component)

Application internals

Inside the single Next.js container: the public pages and staff /edit UI (React Server Components), the API route handlers grouped by purpose — the public-data surface now spans scholars, publications, topics, methods, centers / units, NIH portfolio, funding opportunities, and the center-collaboration network — and the internal libraries they share, wired to Aurora (reader/writer), OpenSearch, the SAML IdP, and the WCM directory (LDAP authz + live headshots). The /edit surface adds Bedrock-backed AI generation (overview · biosketch · CV) — the CV path reads clinical fields live from POPS (lib/edit/pops.ts). middleware.ts gates every /edit path.
