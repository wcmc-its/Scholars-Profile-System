**Issue:** #575 — Accessibility conformance: WCAG 2.2 AA + Section 508 audit before public launch
**Date:** 2026-05-29

## 3. Findings by rule (severity-ranked)

### 3.1 `button-name` — **critical** — WCAG 4.1.2 Name, Role, Value (A) — ✅ fixed
The "Sort by" `Select` triggers on the department faculty / publications / grants tabs render as `role="combobox"` buttons with no accessible name (the `<SelectValue placeholder>` is not an accessible name once a value is selected). A screen-reader user hears an unnamed combobox. Identical pattern was already solved correctly on the topic feed (`aria-label="Show"` / `"Sort by"`).
**Fix:** added `aria-label="Sort by"` to the three department `SelectTrigger`s.

### 3.2 `aria-hidden-focus` — serious — WCAG 4.1.2 (A) — ✅ fixed
Two collapsible regions set `aria-hidden` while keeping their interactive children in the DOM and in the tab order — a keyboard user can Tab into an invisible control:
- The scroll-revealed header search (`scroll-revealed-search.tsx`) when the hero search is still on-screen (home page top).
- The taxonomy callout's "related concepts" list (`taxonomy-callout.tsx`) when collapsed.

**Fix:** added the `inert` attribute (gated to the hidden state) alongside the existing `aria-hidden`. `inert` removes the subtree from the tab order *and* the accessibility tree. Verified empirically with axe on a synthetic node: `aria-hidden` alone → 1 `aria-hidden-focus` violation; `aria-hidden` + `inert` → 0, and the inert child rejects `.focus()`. (React 19 / supported in all current browsers.)

### 3.3 `nested-interactive` — serious — WCAG 4.1.2 (A) — ⏳ backlog
The `/browse` department/center accordion (`browse/departments-grid.tsx`) places interactive elements inside a `<summary>` (28 nodes). A `<summary>` is itself a button, so nesting links/buttons inside it produces an invalid, ambiguous interactive nesting for AT. Needs a small structural change (move the interactive content out of the `<summary>` into the `<details>` body, or replace the `<details>` disclosure with a button + region pattern). Filed as backlog — not a one-line fix.

### 3.4 `target-size` — serious — WCAG 2.5.8 Target Size (Minimum) (**2.2 AA**) — ⏳ backlog
Recurs on profile (24), topic (20), browse (4), center (1). Two offenders:
- The inline **"Copy PMID"** icon buttons (`publication-meta.tsx`) — well under 24×24 CSS px.
- Some pill / chip links (e.g. co-author and member chips) under 24px tall.

This is new in WCAG 2.2 and a genuine AA gap. Remediation is a *visual* change (enlarge the hit area, e.g. via padding or a `::before` expander) and so is **not** applied here per the project rule against unilateral visual/spacing changes — filed as backlog with a proposed approach.

### 3.5 `color-contrast` — serious — WCAG 1.4.3 Contrast (Minimum) (AA) — ⏳ backlog (palette decision)
axe flags 23–93 `incomplete` contrast nodes per page plus several confirmed failures. Recurring offenders: small badge/chip text (`bg-muted` / `bg-[#f…]`), `text-muted-foreground` helper text, and `opacity-70` numerals (citation/grant counts). These touch the **WCM maroon brand palette and the design tokens**, so the fix is a design decision, not a mechanical edit — recorded as a recommendation. A manual contrast sweep with the actual rendered foreground/background pairs should precede any token change.

### 3.6 `landmark-unique` — moderate — WCAG 1.3.1 Info and Relationships (A) — ✅ fixed
The site header's primary `<nav>` had no accessible name, so with the breadcrumb and pagination navs present a screen-reader landmarks list showed multiple indistinct "navigation" regions.
**Fix:** added `aria-label="Primary"` to the header nav.

### 3.7 `heading-order` — moderate — best-practice (supports 1.3.1) — ⏳ backlog
Heading levels skip (e.g. an `<h3>` with no preceding `<h2>`) on `/search` (filter group headings) and `/about` ("Common questions"). Low risk but should be normalized; deferred because it needs per-surface review to pick the right level without changing visual size.

### 3.8 `image-redundant-alt` — minor — best-practice — ✅ fixed
Headshot avatars carried `alt="<scholar name>"` while the same name is always rendered as adjacent visible text, so a screen reader announced the name twice (8–34 nodes per page). Fixed at the single source component (`headshot-avatar.tsx`): the avatar is now `alt=""` (decorative), dropping it from the a11y tree. Verified no test relied on the alt text (`getByAltText` is unused in the suite).

### 3.9 `aria-prohibited-attr` — serious *(incomplete)* — ⏳ backlog (investigate)
axe reported 2 incomplete nodes on the topic page — an ARIA attribute on an element whose role prohibits it. Not pinned to a component in this pass; filed for investigation.

### 3.10 No skip link — WCAG 2.4.1 Bypass Blocks (A) — ✅ fixed
No mechanism existed to bypass the header/nav and jump to content (the first in-page anchor was a "View all N publications" jump, not a skip link).
**Fix:** added a visually-hidden-until-focused "Skip to main content" link as the first focusable element in the public layout, targeting a new `#main-content` wrapper (kept a `<div tabindex="-1">`, not a second `<main>`, since pages render their own `<main>`). On-brand colors reuse the existing design tokens.

- These require either manual testing or an authenticated harness and are out of scope for this automated + structural first pass.

## 8. How to reproduce

The sweep runs axe-core in the browser against each route. To repeat on staging/prod, load axe-core and run with the same tag set:

```js
// In the page console (or via Playwright browser_evaluate), after the page settles:
const r = await axe.run(document, {
  runOnly: { type: 'tag', values: ['wcag2a','wcag2aa','wcag21a','wcag21aa','wcag22aa','best-practice'] }
});
console.table(r.violations.map(v => ({ id: v.id, impact: v.impact, nodes: v.nodes.length })));
```

Routes swept: `/`, `/search?q=cancer`, `/scholars/{slug}`, `/browse`, `/departments/{slug}`, `/centers/{slug}`, `/about`, `/about/feedback`, `/topics/{slug}`.
