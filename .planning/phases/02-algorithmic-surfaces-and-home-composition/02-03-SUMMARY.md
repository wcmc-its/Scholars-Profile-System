**2. `etl/search-index/index.ts` â€” switched `include` to explicit `select`**
- **Found during:** Task 3, when adding `roleCategory: true` to the existing Prisma query
- **Reason:** The original `findMany` used `include` (which pulls all scalar fields by default plus relations). Adding `roleCategory: true` directly to `include` is invalid â€” Prisma requires either an explicit `select` block listing every projected field, or to leave `include` for relations only. Switching to `select` makes the projection explicit and is the conventional Prisma pattern for ETL serialization.
- **Fix:** Refactored to `select: { cwid, slug, preferredName, fullName, primaryTitle, primaryDepartment, overview, roleCategory, topicAssignments, grants, authorships }`. All previously-used scalar fields plus the new `roleCategory` are now explicitly listed.
- **Files:** `etl/search-index/index.ts`
- **Commit:** `f00334c`

These were classified Rule 3 (auto-fix blocking issue) â€” necessary to complete the task; no architectural change required.
